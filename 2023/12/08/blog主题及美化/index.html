blog主题及美化

chen2023-12-082023-12-08

在上一篇中我们已经介绍了如何创建一个hexo博客，但是它实在是太简陋了！所以这一篇我们就来介绍一下如何对我们的博客进行自定义和美化，让他真正成为我们与互联网对接的利器！

在hexo博客中我们既可以通过调整各种源码以及加入脚本进行美化，还可以调用GitHub或者其他开源平台上已经整理好的主题。这里我们可以将两者进行结合，先找到符合我们审美的主题，然后对他进行调整来个性化它

安装主题

hexo博客的主题实在是太多了，我也不可能都用过，所以就介绍给大家我用的感觉比较好的主题

Next主题

Next主题相对于其他国内高度自定义的主题，最大的特点就是简单，它就像一个框架，我们可以在它的基础上进行许许多多自定义的修改

而且符合大多数人的审美也是它成为hexo上使用最多的主题的原因之一

安装

首先，我们来到博客的本地资源文件夹，右键选择Open Git Bash here打开Git Bash

然后我们输入hexo cl来清除静态资源

现在，我们在Git Bash中输入git clone https://github.com/theme-next/hexo-theme-next themes/next将next主题下载到我们本地，这个过程可能有点慢，我们可以按照上一篇blog搭建中开头所说的，使用watt toolkit(steam++)进行加速

如果不出意外的话应该可以在.\themes\next 这个文件夹中看到我们的主题文件了！

完成主题的安装之后，我们会有两个配置文件，我称他们为主题配置文件和博客配置文件，下面第一次提及这两个词的时候我会介绍他们的位置，之后我只用主题配置文件和博客配置文件来说明应该修改的文件

现在，我们来到博客的根目录，用 VS Code 或者类似的软件打开该目录下的**_config.yml**，这个文件既是我说的博客配置文件

然后我们在Extensions下面找到theme，将它的值改为next

还记得我们上一篇说的吗，在yml配置文件语言中，每个语句的冒号后面都要跟空格！！

这时候我们再次去到Git Bash，依次输入hexo cl，hexo g，hexo s，对静态文件进行部署，然后我们来到浏览器，输入localhost:4000，就可以看到我们的next主题了！

next主题有四种风格，分别为muse，mist，prisces，gemini，其中前两个比较相似，后两个比较相似。

我们来到blog文件夹，找到themes>next>_config.yml,这个既是我所说的主题配置文件，打开它，然后找到Scheme Settings语句，就可以看到这四个主题了！next的作者为了方便我们使用，已经列好了它们的选项，我们只需要删除我们想要的主题前面的注释符即可！

这里要注意的是只能有一个scheme语句在使用，也就是说在删掉gemini语句的注释符的时候要记得把muse加上注释

进行客制化

现在我们通过主题对我们简陋的博客进行了初步美化。可是还有一个我们不能忽视的问题是，博客的作者姓名不是我们！这是任何一个博主不能接受的，所以我们接下来就对我们的博客进行客制化，让它真正成为我们的小窝

1.基本信息

现在我们打开博客配置文件，在site语句下就是我们的基本信息了！

值得注意的是，如果我们现在的博客仍在运行中（没有按Ctrl+C 停止本地广播），我们可以在修改过配置之后直接刷新localhost网页来实时预览我们的修改效果。

这里插一句，让搜索引擎爬虫获取我们网站的信息是好事，因为这可以让你完成备案和申请的博客被更多人看到。比如我的blog博客搭建教程就有可能被人百度到，这样我们的分享可以被更多的人看到（然而事实是我的博客既没备案也没申请搜索引擎接入。。。）

OK，修改完成后我们只需要刷新一下localhost网页就可以看到修改了！如果已经使用Ctrl+C 停止了本地广播，可以再次去到Git Bash，输入hexo s来开启本地广播

2.导航栏

我们再次来到主题配置文件，找到menu语句，它对应的就是我们博客导航栏的条目！

我们可以根据需求把相应条目下的注释去掉，比如一般情况下我们需要categories和about，那么我们就把它前面的注释去掉。以categories为例，我们去到Git Bash，输入hexo new page categories（进行此步骤之前请务必使用Ctrl+C 停止本地广播！），这会在我们博客根目录的source文件夹下看到categories文件夹了！我们打开这个文件夹下的index,md文件

在date语句后面另起一行，输入tpye: categories，这时候就完成了。我们保存后可以使用hexo s命令进行查看

不妨测试一下。我们来到博客的根目录，打开source文件夹，找到*_posts文件夹，这是我们所有博文存放的地方。我们可以看到它默认自带的hello world.md文件。这是一篇说明文档，我们打开它，在title语句下面另起一行，输入categories: text，然后*保存

这里我还是想提醒一下，在yml配置文件语言中，每个语句的冒号后面都要跟空格！！

此时我们再次hexo s来到我们的博客，点开categories标签，就可以看到我们刚才设置的text分类了！

同样的，我们在about的index.md文件下输入的字符也都会在about标签下显示

结语

ok，到这里我们的博客美化就初步完成了！如果你想要更进一步的修改博客，比如设置导航栏在右边，将黑色的tag改成别的颜色或者图片，给博客添加背景，修改页脚等等等的操作，请自行查阅主题配置文件。Next主题的作者几乎将每一个语句都进行了注释，让没有任何基础的小白都能够轻松修改，相信对你们来说更不在话下了！

最后提醒一下，各位在修改自己的博客时，一定要先将整个博客文件夹复制到桌面或者其他什么地方进行备份！如果无意间修改了某个选项导致整个博客崩溃无法打开而且还找不到原因，那各位估计也会崩溃！别问我怎么知道的！
