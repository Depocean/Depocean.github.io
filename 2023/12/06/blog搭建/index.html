blog搭建

chen2023-12-062023-12-06

下了不少功夫，也踩了不少坑，博客总算是成型了

OK，废话不多说，我们直接开始博客搭建的教程吧

必要说明

要搭建博客，第一个要准备的就是耐心，你可能会遇到各种各样的问题，这是很正常的。有时候一个问题就要查半天，最后还有可能没解决，但是当问题解决时你会有极大的满足感

注册GitHub

首先，博客作为前端的网页，必须要有资料库，目前最为流行的也是我在使用的是Github的方案

1.打开GitHub官网

如果加载速度过慢，可以使用Watt Toolkit进行加速

2.按照提示一步步进行注册

仓库类型我们选中Choose Free即可，如下图

3.进行登录

注册完就可以进行登录了！期间可能会用到邮箱的验证码，输入即可

进到如图所示的界面，此时我们的Github就注册完成了！

安装Git

我们的博客虽然部署在Github的仓库上，但Github的单文件上传和数据管理仍对我们的博客使用造成了极大的困难，于是我们采用git进行本地数据库的上传

1.下载Git

打开Git官网，点击Download for Windows

选择64-bit Git for Windows Setup进行下载

随后一直点下一步即可，除了更改安装位置(改不改都可以)之外不需要进行任何调整

然后我们在自己电脑上合适的位置新建一个Blog的文件夹 ，在这里我选择的是

各位可自行决定

新建完成后，在文件夹里右键鼠S标，选择

2.完成安装

在git的窗口中，按住Ctrl+滚轮 可以进行缩放。

我们在$ 后面输入git，然后按回车

得到如图所示的代码，那么你的Git已经成功安装了！

绑定Git和GitHub并提交文件

到目前为止你的Git和Github已经完成初步的准备了，但是Git还不知道他要把文件提交到哪里去！所以我们下一步就是要将Git和Github进行绑定，这样才能让本地文件和库牵手

1.生成ssh密钥

我们同样跟上一步一样在新建的Blig文件夹下右键鼠标 ，输入ssh，回车

如果一切正常，你应当会得到如上图所示的代码。如果不一样或者有报错，请自行必应

在Git中，粘贴复制并不是常见的Ctrl+v和Ctrl+c，而是Shift+int(insert)和Ctrl+int(insert)键

然后我们再次输入ssh-keygen -t rsa生成ssh的RSA算法生成密钥，然后我们连续敲四下回车键 。这个操作会在我们电脑中生成两个文件

我们找到名为id_rsa.pub的文件，我们右键以记事本方式打开，将里面所有的代码复制下来

2.进行绑定

我们回到GitHub，选择settings，然后在SSH and GPG keys中添加密钥

然后我们再次回到Git Bash，输入ssh -T git@github.com，一切正常的话，应该会得到如图所示的代码

如果出现如下代码，填入yes即可，这是新用户的提示 ，无需担心

3.提交文件

此时我们的Git已经配置完成，可以开始尝试上传文件了！

我们来到GitHub的主界面，点击右上角的加号，选择New repository，然后按照图示进行配置

这时候就可以看见我们的库了！，我们将鼠标移到绿色的code按钮上，选择SSH，然后将链接复制下来

接下来，我们回到Git Bash，输入git clone 刚才复制的链接，注意clone后面的空格，一切无误的话，我们应该会得到下图所示的文件

此时text已经是一个Git仓库了，现在我们可以开始尝试上传文件了！

我们在这个文件夹下创建一个text,txt文件，内容随意

然后我们先将这个新建的文件提交至缓存库，然后进行提交

来到Git Bash，输入git add text.txt，然后我们使用git commit -m "text"进行提交

如果是第一次使用Git，我们还需要一步额外的操作，我们输入

git config --global user.name "你的GitHub名字，比如我的是Depocean"，回车

git config --global user.email "你的邮箱"，同样回车一次

一定要注意，name后面，引号之前有一个空格！email同样如此

一切无误后，我们输入git push origin main进行提交。第一次上传需要输入密码，在弹出的窗口输入即可

完成后我们去到GitHub，刷新一下应该可以看到我们提交的text文件了！

安装HEXO博客

到现在，我们已经完成了所有的准备工作，是时候开始部署博客了！不过在部署之前，我们还需要一点点调整

1.建立仓库

和上面一样，我们再次来到Github界面，右上角New repository，不过这次不一样的是，我们要将我们的仓库名字

命名为你的名字.github.io

然后，我们来到电脑的本地文件，将上面建立的Blog文件夹清空(可以直接将文件夹删除后再次新建一个)

2.安装nodejs

我们来到node.js的官网，点击左边的版本下载

下载好后打开进行安装，一路默认next即可

安装完成后我们按win键+R ，输入cmd打开

在cmd中，我们分别输入node -v 和npm -v对版本进行检查来验证nodejs是否正确安装，没有问题的话返回结果应该如图

3.配置环境变量

我们找到nodejs的安装目录，如果未被更改的话应该在C:\Program Files\nodejs，如果安装时更改了安装目录就去设定的目录找

在nodejs的目录下新建两个空文件，命名为node_cache和node_global，如图

我们点击win徽标，输入cmd，然后右键以管理员身份运行

在cmd中，我们输入npm config set prefix "C:\Program Files\nodejs\node_global"，此处一定要注意，引号里面的目录是

刚才新建的文件夹的目录，如果安装nodejs时更改了目录，一定要修改这个目录

同样地，我们再次输入npm config set cache "C:\Program Files\nodejs\node_cache"来设置缓存目录

我们打开电脑的设置界面，选择系统>系统信息>高级系统设置

在下方点开环境变量

然后我们在系统变量中新建一个变量名为 “NODE_PATH” ，值为 C:\Program Files\nodejs\node_global\node_modules”

在上面用户变量中，我们双击 Path 路径，点击浏览，选定我们刚才新建的node_global文件夹，如图

然后不要直接点X，从最次一级窗口开始一直点确定，将所有的修改保存

然后我们去到cmd，输入npm install webpack -g，不出意外的话，刚才新建的两个文件夹里面应该已经有文件了

4.安装HEXO

我们去到之间新建的Blog文件夹，鼠标右键 Git Bush Here，然后再Git Bush中输入npm install -g hexo-cli

完成后，我们输入hexo init来初始化文件夹(再次强调，进行这一步之前一定要保证你的Blog文件夹是完全空的！)

没出意外的话，我们输入hexo g进行静态部署

到这一步，我们的hexo博客已经部署完成了，我们可以输入hexo s进行本地部署，此时我们去到浏览器，输入localhost:4000

应该就可以看到我们的网页了！确认无误后，我们回到Git Bash，按Ctrl+C 停止本地广播，否则无法继续输入命令

将Hexo部署到Github

现在，我们的博客已经可以打开了！可是坏消息，只有我们自己能打开！我们还需要将Hexo部署到Github来让其他人也能看到我们的博客。

1.设定地址

一定要注意，在yml数据标记语言中，每个冒号后面都要跟空格！！！！

2.安装git部署插件

然后我们再次去到Git Bash(**注意，一定是在博客文件夹下打开的Git Bash)**，输入npm install hexo-deployer-git --save

来安装git部署插件

完成后我们输入一下三条命令hexo cl hexo g hexo d，他们分别是清除缓存文件，生成静态文件，部署文件到仓库

恭喜！你的博客的初步搭建已经完成了！现在只需要打开浏览器，输入https://xxx.github.io就可以打开你的网站了！
